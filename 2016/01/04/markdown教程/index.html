修辞和强调

Markdown 使用星号和底线来标记需要强调的区段。

效果如下：
Some of these words are emphasized.

Use two asterisks for strong emphasis.

Or, if you prefer, use two underscores instead.

如果你的 * 和 _ 两边都有空白的话，它们就只会被当成普通的符号。

列表

无序列表使用星号、加号和减号来做为列表的项目标记，项目标记后面则一定要接着至少一个空格或制表符。

  • Gum.
  • Booze.

代码

在一般的段落文字中，你可以使用反引号`来标记代码区段

代码区块

如果要建立一个已经格式化好的代码区块，只要每行都缩进 4 个空格或是2个 tab 就可以了

代码区块的效果如下：

这是一个代码区块。

一个代码区块会一直持续到没有缩进的那一行（或是文件结尾）。

代码区块中，一般的 Markdown 语法不会被转换

但是这样做出来的代码区块没有很好看，我查看了hexo中的例子，发现例子中并没有用4个空格或者2个tab的方式来做出代码区块，而是用一下三个反引号！如下：

```
引用的文字
```

如果用了三个反引号，那么引用部分每行前面是有数字的，但是如果用的是四个空格或者两个tab，则是没有数字的，就没有那么好看了。

区块引用

文件中建立一个区块引用，那会看起来像是你自己先断好行，然后在每行的最前面加上>:

这是一个标题。

  1. 这是第一行列表项。
  2. 这是第二行列表项。

给出一些例子代码：

return shell_exec("echo $input | $markdown_script");

引用的区块内也可以使用其他的 Markdown 语法，包括标题、列表、代码区块等，并且可以嵌套。

Markdown 也允许你偷懒只在整个段落的第一行最前面加上 > （PS回车一次和加了空格的效果是一样的，依然是一段！回车两次那就真的分行了）：

This is a block quote with two paragraphs. Lorem ipsum dolor sitamet, consectetuer adipiscing elit. Aliquam hendrerit mi posuere lectus.
Vestibulum enim wisi, viverra nec, fringilla in, laoreet vitae, risus.

自动链接

链接

This is an example link.
I get 10 times more traffic from Google than from
Yahoo or MSN.

title 内容，可以用单引号、双引号或是括弧包着。也可以把 title 放到下一行，也可以加一些缩进，若网址太长的话，这样会比较好看

以下两种方式也行：
